Key Findings

Moltbook takes #1 with a perfect 100/A

The AI agent social network is the only site in the ecosystem with all 7 security headers present. HSTS with preload, full CSP policy, frame ancestors blocked. Agent infrastructure done right.

OpenAI and Anthropic both score C (79/100)

Neither has Content-Security-Policy, X-Frame-Options, X-XSS-Protection, Referrer-Policy, or Permissions-Policy. Only HSTS and X-Content-Type-Options. For companies building the future of AI, the security header hygiene is surprisingly weak.

claude.ai scores lowest among major platforms (62/D)

Despite having 4/7 security headers, claude.ai's low availability score drags it down. The 38ms response time is fast, but the overall scoring penalizes configuration gaps.

GitHub is the fastest (34ms) and second overall

Blazing fast response with 6/7 security headers. Only missing Permissions-Policy. If you're deploying agent services, GitHub Pages is a solid foundation.

UtilShed ranks 17th (70/C) — 0/7 security headers

Yes, we scored our own site. GitHub Pages doesn't set security headers by default, and we haven't configured them via Cloudflare yet. Honest self-assessment. We'll fix this.

Methodology

Each site was checked using the UtilShed Health Check API (free basic tier). The check measures:

  • Availability (50%) — HTTP status, response time (3-sample average), HTTPS enforcement, content type, page size
  • Security Headers (50%) — 7 critical headers: Strict-Transport-Security, Content-Security-Policy, X-Frame-Options, X-Content-Type-Options, X-XSS-Protection, Referrer-Policy, Permissions-Policy

Grades: A (90-100), B (80-89), C (70-79), D (60-69), F (<60). All checks run from Cloudflare Workers edge network.

Data collected March 23, 2026. Scores reflect a point-in-time snapshot and may change as sites update their configurations.
